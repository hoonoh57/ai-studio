## 한 줄 정의

BadaCut은 **"프롬프트 하나로 전문가 수준의 영상이 완성되고, 배포되고, 성장하는"** AI 네이티브 크리에이터 스튜디오다.

---

## 최상위 철학: 결과 먼저, 도구는 나중에

### 벤치마크에서 배운 것

| 제품 | 핵심 설계 | 우리가 가져갈 것 |
|------|-----------|-----------------|
| Suno | Simple → Custom 2단계. 80%가 Simple만 사용 | 입문자에게 프롬프트 한 줄로 완성품 제공 |
| Runway Gen-4 | "Simplicity first, iterate by adding one element" | 단순 시작 → 한 요소씩 추가하는 피드백 루프 |
| Descript | "Edit video like a document" — 타임라인을 숨김 | 복잡한 도구를 필요할 때만 노출 |
| CapCut | Script-to-Video 원클릭 | 스크립트 기반 자동 편집 파이프라인 |
| OpusClip | 긴 영상 → 쇼츠 자동 변환 | 멀티 포맷 자동 리퍼포징 |
| vidIQ/TubeBuddy | SEO 최적화 + 채널 분석 | 배포 후 운영·성장까지 원스톱 |

### 경쟁사 공백 = 우리의 기회

현재 시장에서 크리에이터는 7~10개 도구를 오가며 작업한다:
아이디어(ChatGPT) → 스크립트(Notion) → 편집(Premiere/CapCut) →
음악(Suno) → 썸네일(Canva) → SEO(vidIQ) → 업로드(YouTube Studio) →
분석(TubeBuddy) → 리퍼포징(OpusClip)

**BadaCut은 이 전체를 하나의 프롬프트에서 시작하여 하나의 화면에서 끝낸다.**
이것이 특허 가능한 핵심 가치이며, "이 프로그램 없으면 10배 걸린다"의 근거다.

## 3대 원칙

### 원칙 1: 90% 자동, 10% 사람

AI가 전문가 수준의 결과물을 먼저 만든다.
사람은 만족하면 끝이고, 불만족하면 해당 부분만 터치한다.
자동화 비율이 90% 미만인 기능은 출시하지 않는다.

### 원칙 2: Progressive Disclosure (점진적 공개)

모든 사용자는 같은 진입점(프롬프트 창)에서 시작한다.
복잡성은 사용자가 요청할 때만 한 겹씩 벗겨진다.
기능을 먼저 보여주고 "써보세요"가 아니라,
결과를 먼저 보여주고 "수정할래요?"다.

### 원칙 3: Idea → Publish → Grow 원스톱

영상 "편집"에서 끝나지 않는다.
아이디어 기획 → 제작 → 배포 → 운영/성장까지 전 과정을 커버한다.
크리에이터가 가장 지쳐하는 것은 편집이 아니라 반복 운영이다.

## 5-Phase Pipeline

이것이 BadaCut의 전체 사용자 여정이다.

### Phase 1: IDEATE (아이디어 → 기획)

Copy
사용자 입력: "제주도 3박4일 여행 브이로그"

AI 자동 생성: ├─ 스크립트 (장면별 나레이션 + 예상 러닝타임) ├─ 씬 리스트 (장면 구성표 + 추천 샷 타입) ├─ BGM 추천 (장르/무드/BPM 매칭) ├─ 레퍼런스 패턴 (인기 영상 구조 분석) └─ 채널 맥락 반영 (이전 영상 스타일 자동 참조)

사용자 선택지: [이대로 진행] → Phase 2로 [스크립트 수정] → 자연어로 "인트로 더 짧게" [레퍼런스 변경] → 참고 영상 URL 입력


기획서는 EditorCommand[]로 직렬화되어 Phase 2의 입력이 된다.

### Phase 2: CREATE (소재 → 초안)

사용자 행동: 소재 업로드 (드래그앤드롭 또는 폴더 지정)

AI 자동 처리 (90%): ├─ 소재 자동 분류 (장면/인물/장소 AI 태깅) ├─ 스크립트 기반 자동 컷 편집 ├─ 자막 자동 생성 + 스타일 적용 ├─ BGM 자동 매칭 + 볼륨 덕킹 ├─ 색보정 자동 적용 (채널 스타일 프로필 기반) ├─ 트랜지션 자동 삽입 └─ 모션 프리셋 자동 적용 (등장/퇴장/강조)

사용자 선택지: [미리보기 재생] → 전체 초안 확인 [만족] → Phase 4로 (Phase 3 스킵) [수정할래] → 불만족 영역 선택 → Phase 3


내부적으로 모든 편집 동작은 EditorCommand로 실행된다.
AI가 "brightness +20, fadeIn 0.5s" 같은 명령을 수행하는 것이지,
AI 전용 렌더링 경로는 존재하지 않는다. (R4 유지)

### Phase 3: POLISH (전문가 터치)

진입 조건: 사용자가 "수정할래"를 선택한 경우만

Layer 2 — 컨텍스트 도구: 사용자: "자막을 더 크게" → AI가 자막 크기 자동 조정 사용자: "음악이 시끄러워" → AI가 볼륨 자동 조절 사용자: "여기 트랜지션 바꿔" → 해당 클립의 트랜지션 선택지 표시 → 자연어로 지시하면 AI가 수행. 파라미터 직접 조작 불필요.

Layer 3 — 전문가 파라미터 (요청 시만): 사용자: "키프레임 직접 조정할게" → 해당 클립의 KeyframePanel 오픈 사용자: "이펙트 스택 보여줘" → EffectsPanel이 해당 클립에 대해 열림 사용자: "이전 영상의 색감 적용해줘" → Style Memory에서 프로필 로드

전문가 모드 핵심: ├─ 즐겨찾기 프리셋 라이브러리 (이전 작품에서 원클릭 등록) ├─ A/B 비교 뷰 (이전 영상 vs 새 영상 나란히) ├─ 프리셋 미리보기 (적용 전 실시간 프리뷰) └─ 매크로 스크립트 (반복 작업 자동화)


### Phase 4: PUBLISH (배포)

사용자 행동: "배포" 버튼 클릭

AI 자동 생성: ├─ 썸네일 3종 (AI 생성, A/B 테스트용) ├─ 제목 3안 (SEO 키워드 최적화) ├─ 설명문 (검색 최적화 + 해시태그) ├─ 태그 추출 (콘텐츠 + 트렌드 분석) ├─ 자막 파일 (.srt) 자동 생성 ├─ 멀티 포맷 자동 변환: │ ├─ 16:9 유튜브 본편 │ ├─ 9:16 쇼츠 (하이라이트 3개 자동 추출) │ ├─ 9:16 릴스/틱톡 (플랫폼별 최적화) │ └─ 1:1 인스타그램 피드 └─ 플랫폼별 예약 업로드 설정

사용자 선택지: [썸네일 선택] → 3종 중 택 1 또는 AI 재생성 [제목 선택] → 3안 중 택 1 또는 직접 수정 [예약 시간 설정] → 최적 시간 AI 추천 제공 [배포] → 선택한 플랫폼에 자동 업로드


### Phase 5: GROW (운영/성장)

자동 실행 (배포 후 지속): ├─ 실시간 성과 대시보드 (조회수, 시청 유지율, CTR) ├─ 댓글 감성 분석 + 자동 답글 초안 ├─ A/B 썸네일 성과 비교 → 자동 교체 추천 ├─ 다음 콘텐츠 추천: │ ├─ 채널 트렌드 분석 │ ├─ 경쟁 채널 벤치마크 │ └─ 구독자 관심사 변화 추적 ├─ 시리즈 일관성 관리: │ ├─ 인트로/아웃트로 템플릿 유지 │ ├─ 색감/폰트/음악 스타일 프로필 │ └─ 브랜드 가이드 자동 적용 └─ 월간 채널 성장 리포트

사용자 행동: [다음 영상 만들기] → Phase 1로 (이전 스타일 자동 계승) [프리셋 등록] → 이번 영상의 스타일을 프리셋으로 저장 [채널 전략 조정] → AI와 대화로 방향 조정

## 절대 규칙 (v2 R1~R6 유지 + 추가)

### R7. 결과가 먼저다
어떤 기능이든, 사용자에게 "도구"를 먼저 보여주지 않는다.
AI가 결과를 먼저 생성하고, 사용자는 결과를 보고 판단한다.
"클립을 선택하세요" 같은 빈 화면은 허용되지 않는다.

### R8. 자동화 90% 미달 기능은 출시하지 않는다
사용자가 10단계 중 9단계를 수동으로 해야 하는 기능은 미완성이다.
AI가 9단계를 처리하고 사용자가 1단계만 확인하는 것이 완성이다.

### R9. Phase 스킵이 가능해야 한다
입문자: Phase 1 → (2 자동) → 4 → (5 자동). Phase 3을 모른다.
중급자: Phase 1 → 2 → 3(일부) → 4. 필요한 부분만 터치.
전문가: 어느 Phase든 직접 진입 가능. 자유도 100%.

### R10. 채널 컨텍스트는 누적된다
첫 번째 영상은 프롬프트만으로 만든다.
두 번째 영상부터 AI는 이전 스타일을 기억한다.
열 번째 영상이면 AI는 채널의 정체성을 완전히 이해한다.
이것이 "커피 한 잔 마시면서 이전 수준 이상"의 기술적 근거다.

## 기존 코드의 역할 재정의

현재까지 만든 모든 코드는 폐기하지 않는다. 역할이 바뀐다:

| 기존 코드 | v2 역할 | v3 역할 |
|-----------|--------|--------|
| IconBar + LeftPanel | 사용자가 직접 선택하는 1차 UI | Layer 2~3에서만 노출. Phase 3 진입 시 |
| StudioHub 모달 | 모듈 검색/선택 | Phase 관리자 + 도구 검색 + 프리셋 미리보기 |
| builtinModules | 패널 래핑 | AI Director가 자동 호출하는 백엔드 |
| presetEngine | 수동 프리셋 적용 | AI가 컨텍스트 기반 자동 추천/적용 |
| effectRegistry | 수동 이펙트 추가 | AI가 스크립트 기반 자동 이펙트 체인 구성 |
| editorStore | 수동 상태 관리 | EditorCommand를 통한 AI/수동 통합 상태 관리 |
| KeyframePanel | 수동 키프레임 편집 | Layer 3 전문가 도구. 요청 시만 노출 |

## 특허 가능 영역

1. **Prompt-to-Publish Pipeline**: 자연어 프롬프트 → 완성 영상 → 멀티 플랫폼 배포를 단일 세션에서 처리하는 방법
2. **Channel Style Memory**: 채널의 편집 스타일을 로컬에서 학습하여 새 영상에 자동 적용하는 시스템
3. **Progressive Disclosure Editor**: AI 결과물의 만족도에 따라 편집 도구의 노출 깊이를 동적으로 조절하는 UI 방법
4. **Cross-Video Preset Inheritance**: 이전 영상의 프리셋/스타일을 새 영상에 원클릭으로 계승하는 시스템
5. **Feedback-Loop Refinement**: 자연어 피드백 → AI 재편집 → 미리보기를 반복하여 사용자 만족도를 수렴시키는 방법
