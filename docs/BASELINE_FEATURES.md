## 문서 목적

BadaCut이 CapCut/Premiere Pro/DaVinci Resolve 사용자에게 "이건 물건인데?"라는
첫인상을 주려면, 이들이 이미 손에 익은 **모든 기본 기능**이 최소 동등하거나
더 나아야 한다. 이 문서는:

1. 경쟁사 4종(DaVinci Resolve 20, Premiere Pro, CapCut Desktop, Descript)의
   핵심 편집 기능을 카테고리별로 전수 나열한다.
2. 각 기능별로 경쟁사의 장단점을 분석한다.
3. BadaCut의 현재 구현 상태(✅ 완료 / 🔶 부분 / ❌ 미구현)를 표기한다.
4. 우리가 개선하거나 추가할 사항을 명시한다.
5. 구현 우선순위(B1~B10)와 예상 작업 범위를 확정한다.

### BadaCut 현재 상태: ❌ 치명적 결함

- 오디오가 전혀 재생되지 않음. PreviewArea.tsx에서 video.muted = true 하드코딩.
- Web Audio API 연결 없음. AudioMixerPanel은 UI 껍데기만 존재.
- effectRegistry에 audio-volume, audio-fade 2종이 등록되어 있으나 실제 호출 없음.

### BadaCut 개선/추가 계획

- **B1-1**: Web Audio API AudioContext 생성, 타임라인 클립별 AudioBufferSource 동기 재생
- **B1-2**: 볼륨 슬라이더 실시간 반영 (GainNode)
- **B1-3**: 오디오 페이드 인/아웃 (기존 audio-fade 이펙트 활성화)
- **B1-4**: 볼륨 키프레임 자동화 (기존 keyframe 시스템에 volume 속성 연결)
- **B1-5**: 기본 오디오 미터링 (AnalyserNode → VU 시각화)
- **B1-6**: 오디오 덕킹 자동화 (나레이션 트랙 감지 → BGM GainNode 자동 조절)

### 착수 규칙

1. B1(오디오)과 B2(편집 동작)는 병행 착수 가능 — 의존성 없음
2. B4(자막)와 B5(트랜지션)는 B1 완료 후 착수 — 오디오 동기 필요
3. B7(내보내기)은 B1 완료 후 착수 — 오디오 믹스다운 필요
4. B3, B6, B8, B10은 언제든 병행 가능
5. 각 B 단위는 하위 항목(B1-1~B1-6 등) 순서대로 구현+테스트

### 완료 기준

각 B 항목은 아래 3개를 모두 충족해야 완료:
1. 기능이 실제 동작 (UI + 로직 + 렌더링)
2. 기존 기능과 충돌 없음 (회귀 테스트)
3. CapCut 동급 사용자가 "이거 되네" 수준의 완성도

### 전체 완료 후 다음 단계

B1~B10 전체 완료 → 기존 v3 로드맵의 P4(첫 화면 재설계) 착수
